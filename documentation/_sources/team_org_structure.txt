LEVEL 3 — SQUAD A : PDF EXTRACTION TEAM
================================================================================
  Total Squad Size : 8  (1 lead + 7 contributors)
  Mandate          : Reliably extract clean, structured text from any PDF —
                     whether digital-native, scanned, or mixed — and hand it
                     off to the Translation squad in a normalised format.

  Pipeline Depth   : 2 levels
  ───────────────────────────────────────────
  DEPTH 1 → RAW INGESTION & PARSING
  DEPTH 2 → REFINEMENT & QUALITY ASSURANCE
  ───────────────────────────────────────────

  ════════════════════════════════════════════════════════════
  DEPTH 1 — RAW INGESTION & PARSING
  ════════════════════════════════════════════════════════════

  ┌─────────────────────────────────────────────────────────────────────────┐
  │  SUB-ROLE A1 : PDF PARSING ENGINEERS (3 people)                         │
  │  ─────────────────────────────────────────────────────────────────────  │
  │  Reports to   : PDF Extraction Lead                                     │
  │  Depth Level  : 1 (first pass — raw text extraction)                    │
  │                                                                         │
  │  Primary Tasks                                                          │
  │    · Ingest PDF files via upload API or bulk file-watcher service       │
  │    · Use PDF parsing libraries (PyMuPDF, PDFMiner, Tika, pdfplumber)    │
  │      to extract raw text, preserving reading order                      │
  │    · Detect and extract document structure:                             │
  │        — Headings, paragraphs, lists, footnotes                         │
  │        — Multi-column layouts (split into logical reading flow)         │
  │        — Tables (flag for structured output, not flat text)             │
  │        — Page numbers, headers, footers (tag or strip per config)       │
  │    · Detect document language(s) at page level                          │
  │    · Produce a structured intermediate format (JSON / XML)              │
  │      containing: page number, block type, text content, bounding box   │
  │    · Log parse errors, corrupt files, password-protected PDFs           │
  │    · Maintain file metadata: source name, page count, file size, hash  │
  │                                                                         │
  │  Tooling                                                                │
  │    · Python (PyMuPDF / pdfplumber), Apache Tika, Apache PDFBox         │
  │    · JSON Schema for intermediate format definition                     │
  │    · S3 / GCS for raw file storage                                      │
  │    · Redis queue for ingestion pipeline                                 │
  │                                                                         │
  │  Output Artefact                                                        │
  │    → Raw JSON blocks handed to Depth-2 (OCR & Cleaning)                │
  │      and flagged with parse_confidence score per block                  │
  └─────────────────────────────────────────────────────────────────────────┘

  ┌─────────────────────────────────────────────────────────────────────────┐
  │  SUB-ROLE A2 : OCR SPECIALISTS (2 people)                               │
  │  ─────────────────────────────────────────────────────────────────────  │
  │  Reports to   : PDF Extraction Lead                                     │
  │  Depth Level  : 1 (first pass — for scanned / image-heavy PDFs)        │
  │                                                                         │
  │  Primary Tasks                                                          │
  │    · Identify pages where digital text extraction fails or yields       │
  │      low-confidence output (scanned docs, image-only PDFs)              │
  │    · Pre-process page images before OCR:                                │
  │        — Deskew, denoise, binarise, adjust contrast                     │
  │        — Remove scan artefacts (shadow edges, punch holes)              │
  │        — Detect and correct page rotation                               │
  │    · Run OCR engine(s): Tesseract, Google Vision API, AWS Textract      │
  │    · Handle multi-language OCR (detect script and language per region)  │
  │    · Handle mixed content: text regions + image regions on same page    │
  │    · Produce OCR output in same intermediate JSON schema as A1          │
  │    · Attach confidence score per word / line from OCR engine            │
  │    · Flag low-confidence regions for human review queue                 │
  │    · Maintain OCR engine configuration profiles per document type       │
  │      (forms, books, invoices, certificates, handwritten notes)          │
  │                                                                         │
  │  Tooling                                                                │
  │    · Tesseract 5.x, Google Vision API, AWS Textract, EasyOCR           │
  │    · OpenCV, Pillow for image pre-processing                            │
  │    · Language detection: langdetect, fastText                           │
  │    · GPU-accelerated inference for high-volume OCR jobs                 │
  │                                                                         │
  │  Output Artefact                                                        │
  │    → OCR JSON blocks (same schema as A1) merged into the unified        │
  │      document JSON, flagged with source: "ocr" and confidence score    │
  └─────────────────────────────────────────────────────────────────────────┘

  ════════════════════════════════════════════════════════════
  DEPTH 2 — REFINEMENT & QUALITY ASSURANCE
  ════════════════════════════════════════════════════════════

  ┌─────────────────────────────────────────────────────────────────────────┐
  │  SUB-ROLE A3 : TEXT CLEANING & NORMALISATION ENGINEERS (2 people)       │
  │  ─────────────────────────────────────────────────────────────────────  │
  │  Reports to   : PDF Extraction Lead                                     │
  │  Depth Level  : 2 (second pass — post-extraction refinement)            │
  │                                                                         │
  │  Primary Tasks                                                          │
  │    · Receive merged JSON from Depth-1 (A1 + A2 outputs)                │
  │    · Remove artefacts introduced by parsing or OCR:                     │
  │        — Ligature errors (ﬁ → fi, ﬀ → ff)                              │
  │        — Hyphenation at line breaks (re-join split words)               │
  │        — Spurious newlines mid-paragraph                                │
  │        — Repeated headers/footers across pages                         │
  │        — Page number noise embedded in body text                        │
  │        — Encoding errors (mojibake, garbled Unicode)                    │
  │    · Normalise whitespace, punctuation, quotation marks                 │
  │    · Detect and handle special content:                                 │
  │        — Mathematical expressions (preserve or flag)                   │
  │        — Chemical formulas, code blocks, URLs                           │
  │        — Bullet lists and numbered lists (convert to structured tags)   │
  │    · Re-assemble reading order for multi-column and complex layouts     │
  │    · Strip or preserve formatting metadata per downstream config        │
  │    · Produce the final cleaned-text JSON ready for Translation squad    │
  │                                                                         │
  │  Quality Gate (Depth 2 owns this)                                       │
  │    · Run automated checks: character error rate estimate, word count    │
  │      sanity, empty-block detection, duplicate-page detection            │
  │    · Documents below quality threshold are routed to human review       │
  │      before being passed downstream                                     │
  │    · Maintain a cleaning audit log per document (what was changed, why) │
  │                                                                         │
  │  Tooling                                                                │
  │    · Python (regex, ftfy, unicodedata, BeautifulSoup)                   │
  │    · spaCy / NLTK for sentence boundary detection                       │
  │    · Custom rule engine for domain-specific cleaning profiles           │
  │    · Unit test suite for each cleaning rule (pytest)                    │
  │                                                                         │
  │  Output Artefact                                                        │
  │    → Final extraction_output.json per document                          │
  │      Schema: { doc_id, pages: [{ page_no, blocks: [{ type, text,       │
  │      confidence, language, cleaned: true/false }] }], metadata }        │
  │    → Pushed to Translation squad's input queue                         │
  └─────────────────────────────────────────────────────────────────────────┘

  PDF Extraction Pipeline Summary
  ─────────────────────────────────────────────────────────────────────────
  File In → [A1: Parsing] → [A2: OCR if needed] → [A3: Cleaning & QA]
            (Depth 1)         (Depth 1)              (Depth 2)
  → extraction_output.json → Translation Input Queue
  ─────────────────────────────────────────────────────────────────────────

LEVEL 3 — SQUAD B : TRANSLATION TEAM
================================================================================
  Total Squad Size : 10  (1 lead + 9 contributors)
  Mandate          : Translate extracted text accurately from the source
                     language into target language(s) while preserving
                     document structure, tone, and terminology.

  ┌─────────────────────────────────────────────────────────────────────────┐
  │  SUB-ROLE B1 : NLP / TRANSLATION ENGINEERS (4 people)                  │
  │  ─────────────────────────────────────────────────────────────────────  │
  │  Reports to   : Translation Lead                                        │
  │                                                                         │
  │  Primary Tasks                                                          │
  │    · Integrate and maintain translation engine(s):                      │
  │        — Neural MT: DeepL API, Google Translate API, Azure Translator  │
  │        — Open-source: Helsinki-NLP / Opus-MT, NLLB-200, M2M-100        │
  │    · Build text segmentation pipeline (sentence splitter before MT)     │
  │    · Preserve structural tags during translation (headings, lists)      │
  │    · Implement language routing (detect source → select best model)     │
  │    · Handle domain-specific translation (legal, medical, technical)     │
  │    · Apply pre/post-processing steps around MT engine calls             │
  │    · Cache translated segments to reduce API cost and latency           │
  │    · Support batch translation and streaming (real-time) modes          │
  │    · Monitor translation latency, throughput, and error rates           │
  │                                                                         │
  │  Tooling                                                                │
  │    · Python, HuggingFace Transformers, FastAPI                          │
  │    · DeepL / Google / Azure translation APIs                            │
  │    · Redis for segment-level caching                                    │
  │    · Prometheus + Grafana for pipeline metrics                          │
  └─────────────────────────────────────────────────────────────────────────┘

  ┌─────────────────────────────────────────────────────────────────────────┐
  │  SUB-ROLE B2 : LANGUAGE QUALITY ASSURANCE (3 people)                   │
  │  ─────────────────────────────────────────────────────────────────────  │
  │  Reports to   : Translation Lead                                        │
  │                                                                         │
  │  Primary Tasks                                                          │
  │    · Review samples of translated output for fluency and accuracy       │
  │    · Run automated MT quality metrics: BLEU, COMET, chrF++, TER        │
  │    · Build and maintain a human review queue for low-score segments     │
  │    · Spot-check domain-critical content (legal, medical, financial)     │
  │    · Manage external linguistic reviewers and freelance translators     │
  │    · Feed error patterns back to B1 for model fine-tuning              │
  │    · Maintain per-language quality dashboards                           │
  │    · Write QA test cases for regression (known-difficult sentences)     │
  └─────────────────────────────────────────────────────────────────────────┘

  ┌─────────────────────────────────────────────────────────────────────────┐
  │  SUB-ROLE B3 : TERMINOLOGY & STYLE MANAGEMENT (2 people)               │
  │  ─────────────────────────────────────────────────────────────────────  │
  │  Reports to   : Translation Lead                                        │
  │                                                                         │
  │  Primary Tasks                                                          │
  │    · Build and maintain multilingual glossaries (source ↔ target pairs) │
  │    · Enforce client-specific style guides and brand tone-of-voice rules │
  │    · Integrate terminology into MT pipeline as forced/soft constraints  │
  │    · Manage Translation Memory (TM) database (SDL, memoQ, or custom)   │
  │    · Flag inconsistent terminology usage across documents               │
  │    · Maintain Do-Not-Translate lists (brand names, product codes)       │
  │    · Produce per-project terminology reports                            │
  └─────────────────────────────────────────────────────────────────────────┘

  Translation Pipeline Summary
  ─────────────────────────────────────────────────────────────────────────
  extraction_output.json → [B1: MT Engine] → [B2: LQA] → [B3: Terminology]
  → translation_output.json → TTS Input Queue
  ─────────────────────────────────────────────────────────────────────────

LEVEL 3 — SQUAD C : TEXT-TO-SPEECH (TTS) TEAM
================================================================================
  Total Squad Size : 6  (1 lead + 5 contributors)
  Mandate          : Convert translated text into natural-sounding speech
                     audio files, in the correct language, voice, and format.

  ┌─────────────────────────────────────────────────────────────────────────┐
  │  SUB-ROLE C1 : TTS ENGINE ENGINEERS (3 people)                         │
  │  ─────────────────────────────────────────────────────────────────────  │
  │  Reports to   : TTS Lead                                                │
  │                                                                         │
  │  Primary Tasks                                                          │
  │    · Integrate TTS engine(s):                                           │
  │        — Cloud: Google Text-to-Speech, AWS Polly, Azure TTS            │
  │        — On-prem / open-source: Coqui TTS, Bark, VITS, Piper           │
  │    · Select voice profile per language and document type                │
  │    · Handle SSML (Speech Synthesis Markup Language) for:               │
  │        — Punctuation-driven pausing                                     │
  │        — Emphasis on headings vs body text                              │
  │        — Pronunciation of abbreviations, numerals, dates               │
  │    · Batch-process translated segments into audio chunks                │
  │    · Stitch audio chunks into per-page or per-document audio files     │
  │    · Export in configurable formats: MP3, WAV, OGG, FLAC               │
  │    · Monitor TTS API latency, character cost, and error rates           │
  │    · Implement retry logic and fallback engine routing                  │
  │                                                                         │
  │  Tooling                                                                │
  │    · Python, FFmpeg for audio stitching and format conversion           │
  │    · Google TTS / AWS Polly / Azure TTS SDKs                           │
  │    · Coqui TTS / Piper for self-hosted fallback                         │
  │    · S3 / GCS for audio file storage                                    │
  └─────────────────────────────────────────────────────────────────────────┘

  ┌─────────────────────────────────────────────────────────────────────────┐
  │  SUB-ROLE C2 : AUDIO QUALITY ASSURANCE (2 people)                      │
  │  ─────────────────────────────────────────────────────────────────────  │
  │  Reports to   : TTS Lead                                                │
  │                                                                         │
  │  Primary Tasks                                                          │
  │    · Listen to and rate audio output samples for naturalness, clarity   │
  │    · Run automated audio checks:                                        │
  │        — Silence detection (unexpected gaps or cut-offs)                │
  │        — Clipping / distortion detection                                │
  │        — MOS (Mean Opinion Score) estimation via automated models       │
  │    · Verify correct pronunciation of proper nouns, domain terms         │
  │    · Check sync between text segments and audio timestamps              │
  │    · Flag mispronounced segments back to C1 for SSML correction        │
  │    · Manage listening test panel for multi-language voice validation    │
  │    · Maintain audio QA scorecards per language and voice profile        │
  └─────────────────────────────────────────────────────────────────────────┘

  TTS Pipeline Summary
  ─────────────────────────────────────────────────────────────────────────
  translation_output.json → [C1: TTS Engine] → [C2: Audio QA]
  → final_audio/{doc_id}/{lang}/{page}.mp3 → Delivery / CDN
  ─────────────────────────────────────────────────────────────────────────

  │                                                                        │
  │  INPUT: PDF file(s) uploaded via API or file watcher                  │
  │         ↓                                                              │

  │  └──────────────────────────────────────────────────────────────────┘ │
  │         ↓  extraction_output.json pushed to Translation queue          │

  │  │        → Output: translation_output.json                         │ │

  │  │    C2: Audio QA      → Automated + human voice quality review    │ │
  │  │        → Output: audio files (MP3/WAV) per page / document       │ │
  │  └──────────────────────────────────────────────────────────────────┘ │
  │         ↓  audio files pushed to delivery storage / CDN               │
  │  OUTPUT: Final audio files accessible via API or download link         │
  │                                                                        │
  └────────────────────────────────────────────────────────────────────────┘
